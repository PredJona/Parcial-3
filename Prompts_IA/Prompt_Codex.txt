====================================================
CONTEXTO DEL PROBLEMA
====================================================

Una institución académica desea un sistema para administrar:

- El registro de estudiantes.
- Los cursos en los que participan.
- Las actividades extracurriculares.

Cada estudiante debe poseer como mínimo la siguiente información:

1. Código de estudiante.
2. Nombre completo.
3. Carrera:
   - Ingeniería de Sistemas
   - Administración
   - Contabilidad
   - Derecho
   - Arquitectura
4. Género:
   - Masculino
   - Femenino
5. Modalidad de estudio:
   - Presencial
   - Virtual
   - Semipresencial
6. Cursos:
   - Programación I
   - Base de Datos
   - Redes de Computadoras
   - Sistemas Operativos
   - Administración Financiera
   - Arte Digital
   - Diseño Arquitectónico
   - Contabilidad General
7. Actividades extracurriculares:
   - Deportes
   - Música
   - Arte
   - Investigación

La aplicación debe implementar estas funcionalidades:

1. Registrar estudiantes.
2. Mostrar los estudiantes registrados.
3. Consultar la información almacenada de un estudiante.
4. Eliminar registros seleccionados.
5. Permitir la selección de cursos y actividades mediante controles adecuados.

NO implementar estadísticas básicas. La funcionalidad de estadísticas queda descartada porque se eligieron las opciones 4 y 6 de la guía.

====================================================

====================================================

La interfaz debe ser moderna, clara y agradable, pero debe seguir pareciendo una aplicación Windows Forms académica.

Diseño visual requerido:

- Estilo moderno tipo dashboard académico.
- Colores principales: azul, blanco, violeta o verde suave.
- Buen contraste.
- Tipografía legible.
- Iconos con FontAwesome.Sharp.
- Secciones separadas con paneles o cards.
- Botones claros y alineados.
- Nada desordenado.
- No usar una interfaz sosa o vacía.
- No generar una pantalla demasiado cargada.

Formulario principal sugerido:

FrmRegistroEstudiantes

Debe contener estas secciones:

1. Encabezado superior:
   - Nombre de la app: “EduRegistro”

   - Subtítulo: “Sistema de Registro Académico de Estudiantes”
   - Icono académico.
   - Datos de asignatura, grupo, equipo y usuario/máquina en una zona visible o en pantalla Acerca de.

2. Datos personales:
   - TextBox para código de estudiante.
   - TextBox para nombre completo.
   - RadioButton para género masculino.
   - RadioButton para género femenino.

3. Datos académicos:
   - ComboBox para carrera.

   - ComboBox para tipo de curso si se desea clasificar como Normal/Seminario.

Aunque tipo de curso y jornada no están como datos mínimos en la segunda hoja del problema, pueden incluirse como mejora visual siempre que no afecten los requisitos principales.

4. Cursos inscritos:
   - CheckedListBox o panel con CheckBox para seleccionar varios cursos.
   - Cursos obligatorios:
     - Programación I
     - Base de Datos
     - Redes de Computadoras
     - Sistemas Operativos
     - Administración Financiera
     - Arte Digital

5. Actividades extracurriculares:

     - Deportes

6. Consulta de estudiante:

   - ComboBox para criterio de búsqueda:
     - Código
     - Nombre
     - Carrera
     - Modalidad

   - Consultar estudiante.
   - Mostrar todos.

   - Limpiar campos.

Opcionalmente agregar:

====================================================
REQUISITOS FUNCIONALES
====================================================

La aplicación debe permitir:

1. Registrar estudiantes:
   - Capturar datos desde la interfaz.
   - Validar campos obligatorios.
   - Crear objeto Estudiante.
   - Guardarlo en una lista en memoria.
   - Mostrarlo en el DataGridView.
   - Limpiar campos después de agregar.
   - Evitar códigos duplicados.

2. Mostrar estudiantes registrados:
   - Mostrar todos los estudiantes en el DataGridView.
   - Debe haber datos de prueba opcionales solo para demostración, pero el usuario debe poder ingresar datos reales.
   - Los datos no deben ser únicamente fijos o constantes.

3. Consultar estudiante:
   - Buscar por código, nombre, carrera o modalidad.
   - Mostrar coincidencias en el DataGridView.
   - Si no hay resultados, mostrar MessageBox informativo.
   - Botón “Mostrar todos” debe restaurar la tabla completa.

4. Eliminar registro seleccionado:
   - Seleccionar una fila del DataGridView.
   - Presionar Eliminar.
   - Confirmar con MessageBox.
   - Eliminar el estudiante de la lista.
   - Actualizar tabla.
   - Si no hay fila seleccionada, mostrar advertencia.

5. Selección de cursos y actividades:
   - Debe permitir seleccionar varios cursos.
   - Debe permitir seleccionar varias actividades.
   - Validar que al menos un curso sea seleccionado.
   - Validar que al menos una actividad extracurricular sea seleccionada.

6. Salir:
   - Botón Salir con confirmación.
   - Cierre normal de la aplicación.
